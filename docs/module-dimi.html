(inner) dimi(levelopt, msgFormatopt, serializeopt) → {DimiLogger}

Factory function that returns a DimiLogger object. Dimi is a micro logging utility (dimi is short for diminutive) in that it provides a very small code base for handling logging. If you're looking for a fully-featured log utility check out pino. Dimi is meant for those cases when Pino is too heavy handed.

Parameters:
Name Type Attributes Default Description
level logLevel <optional>
 'info'

The log level to use when writing to the console or stdout (in Node). Defaults to info if the level being passed is not of logLevel.
msgFormat messageFormatFunction <optional>
 defaultMsgFormat
serialize boolean <optional>
 false

If true dimi will serialize metadata using json-stringify-stable to prevent throwing on Circular References, Otherwise, if false metadata will be passed as an Object.

Example
### In Node
```javascript
  const dimi = require('dimi');
  const log = dimi('info', (msg, data, level, time) => {
    // parity the default format function
    return `${time} [$level.toUpperCase()] - ${msg}:\n\n${data}`;
  });
```

### In the Browser
```javascript
  // dimi is defined on the window
  const log = dimi('info', (msg, data, level, time) => {
    // parity the default format function
    return `${time} [$level.toUpperCase()] - ${msg}:\n\n${data}`;
  });
```

DimiLogger :object

The log object that applications use during runtime. A DimiLogger instance only writes to the console or stdout if the level being logged is greater than or equal to the current log level.

DimiLogger levels uses standard integers for it's log levels, e.g.:

  • error is 50
  • warn is 40
  • info is 30
  • debug is 20
  • trace is 10

logFunction()

The log function. All levels share the same function parameters:

  • message ({string}) - The message string to be logged.
  • metadata (metadata) - Object

logFunction will not return a value but will write to the console in a browser environment, or to stdout in Node. Note that trace uses log instead of their namesakes due to default formatting in Node and the Browser.

messageFormatFunction()

The function that formats the log message, metadata, log level, current time, and any other desired log outputs into a string to be written to the console in a browser environment or to stdout in Node. If serialize is set to true on the DimiLogger instance, than metadata will be a serialized string. Dimi uses {@npm json-stringify-stable} to prevent throwing on Circular References

messageFormatFunction is passed the following parameters (in order):

  • message The message that is being logged.
  • metadata The data object (or string if serialize is true).
  • level The level being logged at.
  • time The current time as an ISO8601 String. new Date().toISOString()

metadata :object|array|string|number|boolean

Metadata passed to a logFunction to be logged. Metadata is typically (by default if using the defaultMsgFormat function) serialized. If serialized is set to true on the DimiLogger instance, than metadata is serialized using json-stringify-stable to prevent throwing on Circular References.
